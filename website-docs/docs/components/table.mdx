# Table

import { HyTableComponent } from "@nuralyui/table/react";
import { useState } from "react";

A comprehensive table component that displays structured data with built-in features like sorting, filtering, pagination, row selection, and expandable rows. The table component is designed to handle large datasets efficiently while providing a rich set of interactive features.

## Installation

```bash
npm install @nuralyui/table
```

## Usage

### React

```jsx
import { HyTableComponent } from "@nuralyui/table/react";
function App() {
  const headers = [
    { name: "ID", key: "id" },
    { name: "Name", key: "name" },
    { name: "Status", key: "status" }
  ];
  const rows = [
    { id: 1, name: "John Doe", status: "Active" },
    { id: 2, name: "Jane Smith", status: "Inactive" },
    { id: 3, name: "Bob Johnson", status: "Active" }
  ];
  return (
    <HyTableComponent 
      headers={headers} 
      rows={rows} 
    />
  );
}
```

## When To Use

- **Data Display**: When you need to display structured data in rows and columns
- **Data Management**: For applications requiring sorting, filtering, and pagination of large datasets  
- **Selection Interfaces**: When users need to select single or multiple items from a list
- **Administrative Panels**: For managing users, orders, products, or other business entities
- **Reporting Dashboards**: To display tabular reports and analytics data
- **Content Management**: For organizing and managing content in a structured format

## Props/Attributes

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `headers` | `IHeader[]` | `[]` | Array of header definitions for table columns |
| `rows` | `any[]` | `[]` | Array of data objects to display in table rows |
| `size` | `'small' \| 'normal' \| 'large'` | `'normal'` | Size variant of the table |
| `selectionMode` | `'single' \| 'multiple'` | `undefined` | Enable row selection with specified mode |
| `withFilter` | `boolean` | `false` | Enable search/filter functionality |
| `expandable` | `string` | `undefined` | Column key that can be expanded to show full content |

### IHeader Interface

```typescript
interface IHeader {
  name: string;  // Display name for the column header
  key: string;   // Object key to access data in rows
}
```

### Selection Modes

- **`single`**: Allows selection of only one row at a time
- **`multiple`**: Allows selection of multiple rows with checkboxes

### Size Variants

- **`small`**: Compact table with reduced padding and font size
- **`normal`**: Standard table size (default)
- **`large`**: Expanded table with increased padding and font size

## Events

| Event | Description | Detail |
|-------|-------------|--------|
| `onSelect` | Fired when row selection changes | `{ value: selectedRows[] }` |
| `onPaginate` | Fired when page changes | `{ value: currentPage }` |
| `onSort` | Fired when column sorting changes | `{ value: sortedRows[] }` |
| `onSearch` | Fired when search/filter is applied | `{ value: filteredRows[] }` |

## Built-in Features

### Automatic Pagination

- **Configurable Items Per Page**: Choose from 5, 10, 15, or 20 items per page
- **Page Navigation**: Navigate through pages with previous/next controls
- **Page Indicators**: Shows current page and total pages
- **Smart Updates**: Pagination automatically adjusts when data changes

### Column Sorting

- **Multi-State Sorting**: Click column headers to cycle through ascending, descending, and default order
- **Visual Indicators**: Sort direction is clearly indicated with arrows
- **Type-Aware Sorting**: Automatically handles different data types (strings, numbers, dates)
- **Maintains Selection**: Row selections are preserved during sorting

### Row Actions

- **Selection Feedback**: Visual feedback when rows are selected
- **Bulk Actions Bar**: Appears when multiple rows are selected
- **Cancel Selection**: Easy way to clear all selections
- **Selection Persistence**: Maintains selections across page changes

### Search Functionality

- **Global Search**: Searches across all visible columns
- **Real-time Filtering**: Results update as you type
- **Case-Insensitive**: Search works regardless of text case
- **Exact and Partial Matches**: Finds both exact matches and partial text

## Accessibility

The table component includes comprehensive accessibility features:

- **Keyboard Navigation**: Full keyboard support for navigation and interaction
- **ARIA Labels**: Proper ARIA attributes for screen readers
- **Focus Management**: Clear focus indicators and logical tab order
- **Semantic HTML**: Uses proper table elements (`table`, `thead`, `tbody`, `tr`, `th`, `td`)
- **Selection Indicators**: Clear visual and programmatic indication of selected rows
- **Sort Indicators**: Accessible sort direction indicators for screen readers

## Best Practices

1. **Data Structure**: Ensure consistent data structure across all rows
2. **Header Keys**: Use descriptive and unique keys for header objects
3. **Performance**: For large datasets (>1000 rows), consider server-side pagination
4. **Column Width**: Consider responsive design for mobile devices
5. **Loading States**: Show loading indicators for async data operations
6. **Error Handling**: Provide fallbacks for missing or invalid data
7. **Selection Limits**: Consider disabling selection for very large datasets
8. **Search Optimization**: For large datasets, implement server-side search

## Advanced Usage

### Custom Event Handling

```jsx
function AdvancedTable() {
  const [selectedRows, setSelectedRows] = useState([]);
  const [currentPage, setCurrentPage] = useState(1);
  const handleSelection = (event) => {
    setSelectedRows(event.detail.value);
    // Perform actions with selected rows
  };
  const handlePagination = (event) => {
    setCurrentPage(event.detail.value);
    // Update URL or trigger data loading
  };
  return (
    <HyTableComponent
      headers={headers}
      rows={rows}
      selectionMode="multiple"
      onSelect={handleSelection}
      onPaginate={handlePagination}
    />
  );
}
```

### Dynamic Data Updates

```jsx
function DynamicTable() {
  const [tableData, setTableData] = useState(initialData);
  const refreshData = async () => {
    const newData = await fetchTableData();
    setTableData(newData);
  };
  return (
    <div>
      <button onClick={refreshData}>Refresh Data</button>
      <HyTableComponent headers={headers} rows={tableData} />
    </div>
  );
}
```
